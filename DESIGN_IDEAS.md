Контекст проекта:

       Существующая структура:
       - Django проект с booking и users приложениями
       - Базовая Django admin с минимальной кастомизацией
       - 7 моделей НЕ зарегистрировано: Payment, BookingHistory, PlayerRating, CoachProfile, TrainingSession, Notification, PlayerCoachRelationship
       - Аналитика уже есть в users/analytics.py (get_admin_dashboard_stats, get_player_stats)
       - Chart.js v4.4.0 уже интегрирован

       Требования пользователя:
       1. Подход: Гибрид - Django admin для CRUD + отдельные страницы с современным UI для расписания/аналитики
       2. Расписание (все функции):
         - Недельное расписание всех кортов (табличное представление)
         - Расписание каждого тренера
         - Drag-and-drop редактирование бронирований
         - Быстрое создание бронирований из расписания (клик по пустой ячейке)
       3. Аналитика (все метрики):
         - Финансовые: доход, прибыль, прогнозы
         - Загруженность: корты, тренеры, время
         - Клиенты: активность, удержание, LTV
         - Операционные: бронирования, отмены

       Задача:

       Создай подробный план реализации который включает:

       1. Структуру файлов и приложений
         - Создавать ли отдельное приложение admin_panel или расширить существующие?
         - Структура templates/admin/, static/admin/
         - Где разместить views, urls для кастомных админских страниц
       2. Django Admin улучшения
         - Регистрация всех 7 пропущенных моделей
         - Кастомные ModelAdmin классы с inlines, filters, actions
         - Использовать ли пакеты типа django-admin-tools, grappelli, jazzmin?
       3. Кастомные страницы админа
         - URL структура (/admin-panel/schedule/, /admin-panel/analytics/)
         - Views и templates для расписаний
         - API endpoints для drag-and-drop и быстрого создания
       4. Расписание кортов и тренеров
         - Компонент недельного расписания (HTML/CSS/JS)
         - Как хранить и загружать данные
         - Drag-and-drop библиотека (FullCalendar, SortableJS, или кастом?)

       5. Аналитический дашборд
         - Какие графики создать (используя Chart.js)

         - Фильтры по датам, экспорт данных
       6. Технический стек
         - Frontend: Vanilla JS или минимальный фреймворк?
         - CSS framework (Bootstrap 5, Tailwind, или кастом?)
         - AJAX / Fetch API для интерактивности
       7. Порядок реализации
         - Пошаговый план: что делать сначала, что потом
         - Приоритеты (MVP vs полная версия)
